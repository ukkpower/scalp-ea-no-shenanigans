Smarter Scalping with ADX-Based BarsN Adjustment

June 09, 2025 by Luis

In fast-paced markets, having a dynamic approach to price action can make all the difference. That’s where the ADX-Based BarsN Adjustment feature in our scalping EA comes in—a smart enhancement designed to adapt the EA’s behavior based on market volatility.

What Is ADX-Based BarsN Adjustment?

When enabled, this feature uses the Average Directional Index (ADX) to automatically determine how many bars (BarsN) the EA should look back to identify recent highs and lows in price. Instead of using a fixed number of bars across all market conditions, the EA adapts dynamically: fewer bars in high-volatility conditions and more bars in calmer markets. This helps the strategy stay nimble—reacting quickly when the market is moving fast, and avoiding noise when it's not.

The Settings Breakdown

  • Enabled: Yes
  • ADX Period: 14
  • Timeframe: 1 Hour
  • Volatility Thresholds:
    • Very High: ADX ≥ 50
    • High: ADX ≥ 40
    • Medium: ADX ≥ 30
    • Low: ADX ≥ 20

How BarsN Adjusts by Volatility

ADX Value Range Market Volatility BarsN (Lookback Period)
> 50 Very High 5 bars
> 40 High 10 bars
> 30 Medium 20 bars
> 20 Low 40 bars
≤ 20 Very Low 80 bars

The Code

-   if (Volume[0] > 1 && Close[1] > Open[1])
+   if (Volume[0] > 1 && Close[1] > Open[1] && iRSI(NULL, 0, 14, PRICE_CLOSE, 0) < 70)

Latest Update: Timezone-Based Trading Hours & Modular Code Improvements

Welcome back to the Scalp EA No Shenanigans blog! In this post, I'll walk you through some recent enhancements made to the Expert Advisor to improve flexibility, especially around trading hours based on timezones, and modularizing the code for easier future updates.

What’s New?

  • Added timezone selection for trading hours, allowing you to pick your local timezone or any timezone, with automatic conversion to the broker server time.
  • Refactored trading hours logic into a reusable module, so it’s cleaner and easier to maintain.

Why This Matters

Previously, trading hours were hard-coded in server time, which made it tricky if your broker’s server time was different from your local time zone. This update ensures you can define trading windows in your preferred timezone, and the EA will handle the conversion seamlessly.

Code Changes Overview

Here’s an example snippet showing how the existing hardcoded trading hours logic was replaced with the new modular timezone-based approach.


datetime tradeStart = D'1970.01.01 07:00';
datetime tradeEnd = D'1970.01.01 22:00';

// Use modular timezone conversion to get trading window in server time
datetime tradeStart = GetTradeStartTime(userTimezone, serverTimezone);
datetime tradeEnd = GetTradeEndTime(userTimezone, serverTimezone);

Modular Timezone Trading Hours Functions

The new functions encapsulate all the timezone math, keeping the main EA code clean:


datetime GetTradeStartTime(string userTZ, string serverTZ) {
    // Logic to convert user-defined start time to server time
    // ...
    return serverStartTime;
}

datetime GetTradeEndTime(string userTZ, string serverTZ) {
    // Logic to convert user-defined end time to server time
    // ...
    return serverEndTime;
}

How to Use

Set your desired timezone in the input parameters, for example:

  • Input string userTimezone = "Europe/Dublin";
  • The EA handles the rest, converting your times to broker server time automatically.

Conclusion

This update is just one step toward making Scalp EA more adaptable to diverse trading environments. Stay tuned for more improvements and feel free to provide feedback or requests!

Happy trading!
